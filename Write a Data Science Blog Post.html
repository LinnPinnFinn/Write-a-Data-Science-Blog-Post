Project - Write a Data Science Blog Post¶

In this project, I decided to look at what education or learning sources I should focus on in becoming a data scientist, and how these relate to compensation in the industry. I also explore what programming language I should learn first. To answer these questions, I have used Kaggle's yearly data science & machine learning survey for 2019 to find out what data scientists active in the industry have done and think.

To answer my questions in a structured format I will use the CRISP-DM process, which is a cross industry process for data mining:

1. Business Understanding 2. Data Understanding 3. Prepare Data 4. Data Modeling 5. Evaluate the Results 6. Deploy

1. Business Understanding¶

As an anspiring data scientist myself, I am interested in understanding what education or learning sources I should focus on to break into the industry, and how these relate to the compensations in the industry. I would also like to understand where my focus should be in terms of programming languages, and to explore this I have four questions I would like to answer:

Business Questions:

  1. What education or learning platforms should I focus on in becoming a Data Scientist?
  2. How do different education levels and learning platforms relate to compensation?
  3. What programming language should I learn?
  4. What biases might respondents have in recommending which programming language to learn?

2. Data Understanding¶

Below we see the survey methodology from Kaggle and two separate dataframes with all the questions and a survey schema that explains which respondents that have been excluded from certain questions.

Survey Methodology from Kaggle:

  • This survey received 19,717 usable respondents from 171 countries and territories. If a country or territory received less than 50 respondents, we grouped them into a group named “Other” for anonymity.
  • We excluded respondents who were flagged by our survey system as “Spam”.
  • Most of our respondents were found primarily through Kaggle channels, like our email list, discussion forums and social media channels.
  • The survey was live from October 8th to October 28th. We allowed respondents to complete the survey at any time during that window. The median response time for those who participated in the survey was approximately 10 minutes.
  • Not every question was shown to every respondent. You can learn more about the different segments we used in the survey_schema.csv file. In general, respondents with more experience were asked more questions and respondents with less experience were asked less questions.
  • To protect the respondents’ identity, the answers to multiple choice questions have been separated into a separate data file from the open-ended responses. We do not provide a key to match up the multiple choice and free form responses. Further, the free form responses have been randomized column-wise such that the responses that appear on the same row did not necessarily come from the same survey-taker.
  • Multiple choice single response questions fit into individual columns whereas multiple choice multiple response questions were split into multiple columns. Text responses were encoded to protect user privacy and countries with fewer than 50 respondents were grouped into the category "other".

3. Prepare Data¶

In order to prepare our data for modelling, we need to:

  • remove columns with text responses, they have no response in them
  • only keep survey data where the respondent is active as a data scientist in the industry
  • recode our target variable ‘yearly compensation in USD’ (Q10) into one that can be used in modelling. We will make it numerical and ordinal based on the compensation brackets
  • recode one of our input features ‘on which platforms respondents have begun or completed data science courses’ (Q13) into a format that can be used in modeling. We will rename each column header to represent the platform referred to, and make them numerical binary columns
  • one hot encode categorical input feature ‘highest level of education attained or planned to attain within 2 years’ (Q4) to make it into numerical binary columns per category option that can be used in modelling
  • split our data it into training and test sets to use to train and test our models
  • create a new dataframe with percentage of respondents that recommend to learn a programming language and use it themselves regularly, the percentage of respondents who recommend it but don’t use it and the difference between these groups. To do this, we need to create new columns where respondents are recommending the programming language in Q19 and stated that they use it on a regular basis in Q18, and respondents that recommend it in Q19 but not stated that they use it in Q18

def train_models(X_train, y_train):
    '''
    Function that:
    - instantiates and trains regression models on train sets
        
    INPUT:
    'X_train' - train set
    'y_train' - train set
    
    OUTPUT:
    trained regression models:
     - 'lr_model': linear regression model
     - 'rr_model': ridge regression model
     - 'er_model': elastic-net regression model
     - 'br_model': baynesian ridge regression model
    '''
    # instantiate and train baseline regression models

    # linear regression
    lr = LinearRegression(normalize=True)
    lr_model = lr.fit(X_train, y_train)

    # ridge regression
    rr = linear_model.Ridge(normalize=True)
    rr_model = rr.fit(X_train, y_train)

    # elastic-net regression
    er = ElasticNet(normalize=True, random_state=4)
    er_model = er.fit(X_train, y_train)

    # baynesian ridge regression
    br = linear_model.BayesianRidge(normalize=True)
    br_model = br.fit(X_train, y_train)

def calc_coefs(feature_names, model_coefficients):
    '''
    Function that:
    - creates new dataframe with coefficients per input feature from best tuned model, recodes these into 
      new absolute values in new column
    - scales coefficients by using a minmax scaler to be in the range of -1 to 1, merges this column onto
      dataframe and sets feture names as index
        
    INPUT:
    'feature_names': input feature names
    'model_coefficients': coefficients for model
    
    OUTPUT:
    coef_df (pandas dataframe)
    '''
    # calculate coefficient estimates per input feature for best model

    # create new dataframe
    coef_df = pd.DataFrame()
    
    # create column with feature names
    coef_df['est_int'] = feature_names
    
    # create column with feature coefficients
    coef_df['coefs'] = model_coefficients
    
    # create column with feature coefficients as absolute values
    coef_df['abs_coefs'] = np.abs(model_coefficients)
    
    # sort dataframe by 'abs_coefs' in descending order
    coef_df = coef_df.sort_values('abs_coefs', ascending=False)
    
    # reset index to merge on column with scaled features
    coef_df = coef_df.reset_index(drop=True)

    # normalize coefficient weights before plotting them

    # store 'coefs' as 'x'
    x = coef_df[['coefs']]

    # create minmaxscaler object
    scaler = MinMaxScaler(feature_range=(-1, 1))

    # fit and transform 'x' to minmaxscaler object
    x_scaled = scaler.fit_transform(x)

    # create dataframe of 'x'
    coef_normalized = pd.DataFrame(x_scaled, columns=['scaled_coefs'])

    # merge 'coef_df' and 'coef_normalized' on index of both the dataframes
    coef_df = coef_df.merge(coef_normalized, how='left', left_index=True, right_index=True)

    # set 'est_int' as index
    coef_df = coef_df.set_index('est_int')

    return coef_df
